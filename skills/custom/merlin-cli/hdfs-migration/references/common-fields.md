# 公共字段与澄清规则

这份文档用于统一解释 HDFS 跨洋传输任务中的公共字段。开始任何操作前，先确认当前需求究竟是“发起、查看、停止、重试”中的哪一种，再按需收集字段。

## 常见字段

| 字段 | 含义 | 常见来源 |
|------|------|------|
| `source_path` | 源 HDFS 路径 | 用户描述、任务配置 |
| `target_dc` | 目标 HDFS 机房 | 用户描述、任务配置 |
| `target_path` | 目标 HDFS 路径 | 用户描述、任务配置 |
| `task_id` | 传输任务本身的任务 ID | 任务详情、任务列表、链接 |
| `tiansuan_task_id` | 传输任务下子任务对应的 Tiansuan 任务 ID | 子任务详情、执行记录 |
| `zeus_task_id` | 传输任务下子任务对应的 Zeus 任务 ID | 子任务详情、执行记录 |
| `status` | 任务状态 | 查询结果 |
| `creator` | 任务创建人 | 查询结果、筛选条件 |

## 常见术语映射

用户可能不会使用标准术语，要做语义归一化：

| 用户说法 | 可映射为 |
|------|------|
| 跨洋同步、跨地域同步、机房间同步 | HDFS 到 HDFS 的迁移任务 |
| 搬迁、搬数据、回传、复制一份 | 发起任务 |
| 看下这个同步、查一下这个任务 | 查看任务 |
| 停掉这个同步、取消这条传输 | 停止任务 |
| 失败了重来一次、重新传、重跑一下 | 重试任务 |

## ID 字段约定

为了避免混淆，统一使用以下命名：

- `task_id`：表示传输任务本身的 ID
- `tiansuan_task_id`：表示该传输任务下子任务对应的 Tiansuan 任务 ID
- `zeus_task_id`：表示该传输任务下子任务对应的 Zeus 任务 ID

当用户说“任务 ID”但上下文不清楚时，先确认他说的是传输任务本身，还是底层子任务使用的 Tiansuan / Zeus 任务。

## 各操作的最小必要信息

| 操作 | 最少需要的信息 |
|------|------|
| 发起任务 | `source_path`、`target_path`；`target_dc` 优先通过 `target_path` 推断，只有无法唯一推断时再补问 |
| 查看任务 | `task_id` 或 `task_id` 列表；如果没有，则至少给筛选条件，如创建人、时间范围、状态、HDFS 源路径 |
| 停止任务 | `task_id` 或 `task_id` 列表 |
| 重试任务 | `task_id` 或 `task_id` 列表；如果都没有，则至少提供一组能唯一定位历史任务的条件 |

## 补问策略

只问会影响执行的最小问题，避免一次性盘问太多细节。

推荐顺序：

1. 先确认操作类型
2. 再补这次操作的最小必要字段

示例：

- 发起任务但没有目标信息时：请补充目标路径；如果目标机房无法从目标路径唯一推断，再补充目标机房
- 用户说“帮我看下这个任务”但没给 ID 时：请提供任务 ID，或者给我创建人、时间范围和状态，我帮你筛
- 用户说“停掉昨天那个同步”时：先让用户提供任务 ID，或先帮他列出昨天的任务再确认

## 不要猜测的内容

以下字段不要替用户猜：

- 目标机房
- HDFS 路径

## 高风险动作

涉及以下情况时，执行前要再次确认：

- 停止正在运行的任务
- 重新发起可能产生重复数据的重试任务
- 对多条候选任务中的某一条执行停止或重试

## 结果呈现建议

回复时尽量结构化，至少包含：

- 操作类型
- 已识别信息
- 缺失信息
- 执行结果或下一步
