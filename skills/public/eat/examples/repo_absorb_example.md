# Example: absorb a repo into reusable capability

## User Input

喂你一个 GitHub 项目，帮我吸收成能力，并判断要不要沉淀成 skill：
https://github.com/example/project

重点看：
- 它的目录结构为什么这样设计
- 哪些工作流适合并入我们现有技能库
- 最后给我一个可执行的落库建议

## Ideal Output Shape

### 吸收结论
- 该仓库本质上解决了什么问题
- 最值得借鉴的设计是什么

### 关键知识点
- 模块划分
- 执行入口
- 配置模式
- 工作流抽象

### 建议落库方式
- 写到哪个 skill
- 写到哪个文件
- 是否值得新增脚本

- 继续生成 skill skeleton
- 继续生成示例脚本
