# 吸收模板

当你分析一个知识源时，优先按下面的结构记录：

## 1. Source Profile
- 来源类型：repo / article / post / image / doc / mixed
- 主题：
- 可信度：高 / 中 / 低
- 值得吸收指数：1~5

## 2. Problem & Value
- 它解决什么问题？
- 为什么值得现在吸收？
- 对现有 agent / 技能体系有什么补益？

## 3. Core Mechanism
- 核心流程
- 核心抽象
- 输入输出
- 关键约束
- 失败模式

## 4. Reusable Assets
- 可复用 Prompt
- 可复用命令
- 可复用脚本骨架
- 可复用目录结构
- 可复用 checklist

## 5. Skillification
- 应否形成新技能？为什么？
- 如果并入已有技能，应并入哪里？
- 建议写入的文件：SKILL.md / EXTEND.md / references / scripts / examples

## 6. Actionables
- 立刻可做的实现动作
- 需要补充的信息
- 潜在风险
